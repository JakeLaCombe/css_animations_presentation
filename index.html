Let's start with simplicity

Transitions

A transition is an animation that exaggerates a style change to an element.

It's something like this.

And all you need is this!

						
	.demo-link {
          display: inline-block;
          transition: transform 4s !important;
	}

	.demo-link:hover {
          transform: rotate(360deg) !important;
	}

Let's dive Deeper into this

Transition: (property) (duration) (timing_function) (delay)

Property:
What property you aim to transition
Duration:
How long should the transition last
Timing Function:
How should the animation be eased (ease, linear, ease-in, ease-out)
Delay:
What should the animation delay look like.

Want to have fun in your app?

							
  * {
    transition: all 2s;
  }

Why do this instead of JavaScript?

Transition effects are concerned more with styling than application logic.

Thus, this should be placed inside your stylesheets, and not in your JavaScript file.

So much easier to write as well!

Note how this lets us keep your styles inside the CSS file.

							
$(document).on('mouseover', '.transitied-link', function(element) {
  $(element).fadeTo(0, 2);
})

$(document).on('mouseout', '.transitied-link', function(element) {
  $(element).fadeTo(1, 2);
});
							
							
		.transitied-link {
		  opacity: 1
		  transition: opacity 2s;
		}

		.transitied-link:hover {
		  opacity: 0;
		}

Limitations

Transitions can not do anything on page load. You need to go somewhere else, or use a little javascript to use CSS transitions on page load.


You have no logic control over the transition. It's an all or nothing when it comes to using them. (You can't stop in between for example).


If you need a large animation with lots of behavior, transitions will not be able to assist you here. (Anything that uses more than one style change)

CSS3 Animation

CSS3 Animation

A CSS3 Animation is what it sounds like - You use CSS to animate an element on the page

No JavaScript needed - Pure CSS

							
								animation: name duration timing-function delay iteration-count direction fill-mode play-state;
Name:
Name of the animation
Duration:
How long the animation lasts for
Timing Function:
How you want the animation to progress
Delay:
When to start the animation
Iteration Count:
How many times you want the animation to occur.
Direction:
Whether to reverse the animation when an animation is complete
Fill Mode:
What values the element will take when the animation is complete

The Name Property

The name property is a animation definition that consists of a listing of different keyframes

Keyframes Syntax

							
                @keyframes (name) {
                   from { /* Insert from Styles here */}
                   to { /* Insert two Styles here */}
                }

                @keyframes (name) {
                   10% { /* Insert Styles here */ }
                   30% { /* Insert Styles here */ }
                   60% { /* Insert Styles here */ }
                   65% { /* Insert Styles here */ }
                   85% { /* Insert Styles here */ }
                  100% { /* Insert Styles here */ }
                }

The full Example

							
     .animate-paragraph {
	background-color: #222222;
	color: #fff;
	position: absolute;
	left: 25px;
	top: 25px;
	animation: movearound 5s infinite;
     }

    @keyframes movearound {
      10% {
        left: 60px;
      }

      30% {
        left: 100px;
      }

      50% {
        opacity: 0;
      }

      70% {
        top: 100%;
      }

      100% {
        top: 25px;
        left: 25px;
      }
    }
